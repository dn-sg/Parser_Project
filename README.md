# Parser_Project
HSE project
## Описание проекта
### Что будет видеть пользователь
Мы планируем создать сайт на котором будут присутствовать

• Дашборд с количеством собираемых данных за промежуток времени (день/час)

• Статус парсеров (таблица с url сайта и статусом парсера)

• Таблица с данными со всех сайтов. Будет присутствовать возможность
сортировки/фильтрации, открытии информации в модальном окне, скачивания таблицы в
csv формате

• Таблица с запущенными и выполненными задачами по парсингу и их статусы (ошибки
тоже здесь)

### Какие задачи необходимо выполнить
1. Docker Compose и проектирование бд
  o Docker Compose
    ▪ WEB
    ▪ Database
    ▪ Celery
    ▪ Flower
  o БД
    ▪ Таблица Source
    ▪ Таблица Parsed_data
    ▪ Таблица Logs
3. Создание парсеров и их тестирование
Выберем несколько сайтов, создадим для них парсеры. Данные будем записывать в
таблицу Parsed_data; статус парсера, время запуска, код ошибки в таблицу Logs
4. Создание API для фронтенда
Get для фронта
5. Создание фронтенда
o /dash
▪ Дашборд
o /status
▪ Таблица с url / status_parser
o /data
▪ Сводная таблица данных со всех сайтов и возможностью их
фильтрации/сортировки/просмотра/скачивания
o /tasks
▪ Таблица с историей задач по парсингу
6. Общее тестирование проекта
Сколько времени потребуется на каждую задачу
• 1 этап – 3 дня
• 2 этап – 7 дней
• 3 этап – 3 дня
• 4 этап – 10 дней
• 5 этап – 3 дня
